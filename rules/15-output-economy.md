# Output economy

- Lead with the outcome; do not repeat known context.
- Use confirmed business terms, roles, and UI labels before code identifiers.
- Include files, APIs, schemas, or commands only when requested or needed to decide, debug, reproduce, or verify.
- Put observable impact before technical detail.
- Use a table or headings for three or more related items; avoid file-list dumps.
- Report status with concise evidence in natural Vietnamese, except identifiers and code.
