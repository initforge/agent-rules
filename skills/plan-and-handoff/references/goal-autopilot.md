# `/goal` automatic execution

`/goal` creates a scaled executable plan, then an explicit execute pivot starts automatic delivery. The main agent keeps intent and final review, allocates disjoint work through context capsules, and continues dependency-ready slices without owner phase relay.

For long/resumable work, create the detailed ledger before delegation and map source requirements plus later injections to slices. For ordinary work, use the smallest plan and no ledger unless it adds real continuity or proof value.

Never use a default Stop loop to simulate persistence. Native lifecycle facilities may assist where proven; otherwise report their limitation truthfully and rely on the portable plan/ledger plus normal agent behavior.
