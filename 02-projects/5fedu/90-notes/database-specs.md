# ĐẶC TẢ CƠ SỞ DỮ LIỆU SQL DDL – NOSTIME (CHUẨN 5FEDU)

Tài liệu này lưu trữ toàn bộ các mã lệnh SQL DDL để khởi tạo cấu trúc cơ sở dữ liệu trên Supabase khi dự án chuyển sang giai đoạn kết nối thật. 

Mọi bảng nghiệp vụ đều tuân thủ các nguyên tắc:
1. Khóa chính `id int8 generated by default as identity primary key`.
2. Khóa ngoại `id_*` kiểu `int8`.
3. Có cột audit `tg_tao`, `tg_cap_nhat`, `id_nguoi_tao`.
4. Trigger cập nhật `tg_cap_nhat` tự động trên sự kiện `BEFORE UPDATE`.
5. Chính sách RLS đơn giản `authenticated_all` (cho phép toàn bộ tài khoản đăng nhập CRUD, việc lọc dữ liệu xử lý ở client-side).

---

## 1. HÀM VÀ TRIGGER DÙNG CHUNG (SHARED TRIGGERS)

```sql
-- 1. Hàm tự động cập nhật cột tg_cap_nhat
CREATE OR REPLACE FUNCTION public.handle_update_timestamp()
RETURNS TRIGGER AS $$
BEGIN
    NEW.tg_cap_nhat = now();
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

## 2. PHÂN HỆ WEBSITE & MARKETING (Prefix `web_`)

### Bảng 1: `web_thuong_hieu` (Thương hiệu - Brand)
```sql
CREATE TABLE public.web_thuong_hieu (
    id int8 generated by default as identity primary key,
    ten text not null,
    logo_url text,
    mo_ta text,
    trang_thai text default 'Đang hoạt động',
    tg_tao timestamp default now(),
    tg_cap_nhat timestamp default now()
);

-- Trigger cập nhật tg_cap_nhat
CREATE TRIGGER tr_web_thuong_hieu_cap_nhat
BEFORE UPDATE ON public.web_thuong_hieu
FOR EACH ROW EXECUTE FUNCTION public.handle_update_timestamp();

-- RLS Policy
ALTER TABLE public.web_thuong_hieu ENABLE ROW LEVEL SECURITY;
CREATE POLICY "authenticated_all" ON public.web_thuong_hieu FOR ALL TO authenticated USING (true) WITH CHECK (true);
```
