# Knowledge system

Context loads progressively: bootstrap → nearest repository files → one triggered capability → required references → project index/domain pack → external sources. Raw evidence and generated outputs never load by default.

Capability trigger ownership lives in each `SKILL.md` description. The core contains the activation algorithm, not a duplicate registry. Project knowledge has one canonical `context/` tree; platform adapters point to it.

Budgets: bootstrap 300 tokens, core 4,000 total, overlay 600/platform, base capability 3,500, project index 1,800 and lazy project domain pack 8,000. Files above the old 1,500-pack target must remain behind routing indexes and must not be copied into always-loaded project adapters.
